# Product

## Назначение

Создать 1D программный комплекс для нефтяной геомеханики, который помогает инженеру построить механическую модель скважины по глубине, оценить напряженное состояние, поровое давление, прочность пород и безопасное окно бурового раствора.

Продукт должен быть удобен для быстрых инженерных сценариев: загрузить каротаж, настроить параметры, получить профили напряжений, интерпретировать риски и сформировать отчет.

## Целевая аудитория

- геомеханики нефтегазовых компаний;
- инженеры по бурению;
- петрофизики и геологи, готовящие входные данные;
- исследовательские команды, которым нужен прозрачный расчетный прототип.

## Основные сценарии

1. Загрузка LAS/CSV/Excel данных по скважине.
2. Проверка качества входных кривых и единиц измерения.
3. Расчет вертикального напряжения `Sv`.
4. Оценка порового давления `Pp`.
5. Расчет эффективных напряжений.
6. Оценка минимального и максимального горизонтальных напряжений `Shmin` и `SHmax`.
7. Построение профилей механических свойств пород.
8. Расчет окна бурового раствора.
9. Анализ рисков: breakouts, tensile failure, kick/losses, narrow mud window.
10. Экспорт графиков, таблиц и инженерного отчета.

## Функциональные требования

### Импорт данных

- FR-001: приложение должно импортировать LAS 2.0/3.0 файлы.
- FR-002: приложение должно импортировать CSV и XLSX таблицы.
- FR-003: пользователь должен сопоставлять входные кривые с внутренними именами: `DEPT`, `TVD`, `RHOB`, `DTC`, `DTS`, `GR`, `NPHI`, `RES`, `PP`, `UCS`, `PHI`.
- FR-004: приложение должно хранить исходные единицы и приводить расчеты к внутренней SI-системе.
- FR-005: приложение должно показывать пропуски, выбросы и интервалы недостоверных данных.

### Подготовка данных

- FR-010: пользователь должен выбирать интервал расчета по глубине.
- FR-011: приложение должно поддерживать ресемплинг по регулярному шагу.
- FR-012: приложение должно поддерживать интерполяцию небольших пропусков.
- FR-013: приложение должно запрещать расчет при немонотонной глубине.
- FR-014: приложение должно сохранять все примененные операции подготовки данных.

### Расчетная геомеханика

- FR-020: приложение должно рассчитывать `Sv` интегрированием плотности по TVD.
- FR-021: приложение должно поддерживать расчет нормального гидростатического давления.
- FR-022: приложение должно поддерживать минимум один метод оценки аномального порового давления.
- FR-023: приложение должно рассчитывать эффективные напряжения с учетом коэффициента Биота.
- FR-024: приложение должно рассчитывать упругие свойства из акустики и плотности при наличии данных.
- FR-025: приложение должно поддерживать расчет `Shmin` и `SHmax` через выбранную модель напряжений.
- FR-026: приложение должно рассчитывать критические давления разрушения стенки скважины.
- FR-027: приложение должно рассчитывать нижнюю и верхнюю границы безопасного окна бурового раствора.

### Калибровка

- FR-030: пользователь должен вводить LOT/FIT/XLOT точки.
- FR-031: пользователь должен вводить MDT/RFT/pressure test точки.
- FR-032: пользователь должен вводить наблюдения по кавернометрии и breakouts.
- FR-033: приложение должно сравнивать расчетные профили с точками калибровки.
- FR-034: пользователь должен видеть, какие параметры были откалиброваны вручную.

### Визуализация

- FR-040: приложение должно отображать треки по глубине.
- FR-041: пользователь должен включать/выключать расчетные кривые.
- FR-042: приложение должно показывать окно бурового раствора в ppg, SG и MPa.
- FR-043: приложение должно визуально выделять зоны риска.
- FR-044: графики должны экспортироваться в PNG/PDF.

### Отчеты и экспорт

- FR-050: приложение должно экспортировать расчетные кривые в CSV/XLSX.
- FR-051: приложение должно формировать краткий инженерный отчет.
- FR-052: отчет должен содержать входные данные, допущения, методы, графики, предупреждения и версию расчета.
- FR-053: проект расчета должен сохраняться и повторно открываться.

## Нефункциональные требования

- NFR-001: все расчеты должны быть воспроизводимыми.
- NFR-002: каждая расчетная функция должна иметь unit tests.
- NFR-003: ключевые модели должны иметь benchmark cases.
- NFR-004: расчет для одной скважины до 10 000 м с шагом 0.1 м должен выполняться интерактивно.
- NFR-005: код расчетного ядра должен быть отделен от UI.
- NFR-006: приложение не должно молча менять единицы измерения или соглашения по знакам.
- NFR-007: любые NaN, clipping, interpolation и smoothing должны быть явно отражены в audit trail.
- NFR-008: формулы и допущения должны быть документированы в `docs/physics-model.md`.

## MVP

MVP считается готовым, если пользователь может:

1. Загрузить один LAS/CSV файл.
2. Выбрать кривые глубины и плотности.
3. Рассчитать `Sv`, hydrostatic `Pp`, effective stress.
4. Задать простые параметры прочности пород.
5. Получить базовое mud weight window.
6. Увидеть графики по глубине.
7. Экспортировать таблицу результатов.
8. Прогнать тесты и benchmark cases.

## Критерии успеха

- расчеты совпадают с синтетическими эталонами в пределах заданного допуска;
- все единицы и знаки явно задокументированы;
- новый метод можно добавить без переписывания UI;
- пользователь может объяснить, откуда взялась каждая расчетная кривая;
- проект можно безопасно развивать двумя AI-агентами через задачи, тесты и review.
