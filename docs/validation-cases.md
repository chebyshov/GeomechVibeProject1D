# Validation Cases

## Назначение

Validation cases нужны, чтобы любой агент или разработчик мог менять код и быстро понимать, не сломал ли он физику расчета.

## Обязательные синтетические кейсы

### Case 001: Constant Density

Цель: проверить интегрирование вертикального напряжения.

Вход:

- `TVD = 0..4000 m`;
- `RHOB = 2.30 g/cm3`;
- no water depth.

Ожидание:

- `Sv` линейно растет с глубиной;
- `Sv(4000 m) ~= 90.22 MPa`;
- допуск: `0.1 MPa`.

### Case 002: Hydrostatic Pore Pressure

Цель: проверить расчет гидростатического давления.

Вход:

- `TVD = 0..4000 m`;
- `rho_fluid = 1000 kg/m3`.

Ожидание:

- `Pp(4000 m) ~= 39.23 MPa`;
- допуск: `0.05 MPa`.

### Case 003: Effective Stress

Цель: проверить соглашение по эффективным напряжениям.

Вход:

- `Sv = 90 MPa`;
- `Pp = 40 MPa`;
- `alpha = 1.0`.

Ожидание:

- `sigma_v_eff = 50 MPa`.

### Case 004: Missing Density Interval

Цель: проверить интерполяцию и audit trail.

Вход:

- короткий пропуск `RHOB` внутри интервала;
- разрешенная интерполяция до заданной длины пропуска.

Ожидание:

- расчет выполняется;
- warning записан;
- audit trail содержит интервал интерполяции.

### Case 005: Non-Monotonic Depth

Цель: проверить блокировку некорректной глубины.

Вход:

- `TVD` содержит обратный шаг.

Ожидание:

- расчет не выполняется;
- возвращается blocking error.

### Case 006: Narrow Mud Window

Цель: проверить выявление зоны риска.

Вход:

- профиль, где нижняя граница mud window близка к верхней.

Ожидание:

- интервал помечается `NARROW_WINDOW`;
- ширина окна считается в SG/ppg/MPa.

## Реальные или обезличенные кейсы

Добавлять только после согласования:

- West Siberia synthetic well;
- high pore pressure case;
- depleted reservoir case;
- weak shale interval case;
- carbonate high UCS case.

## Правила benchmark-файлов

- каждый case хранится в `benchmarks/`;
- каждый case имеет входной файл, ожидаемый результат и README;
- допуски фиксируются явно;
- benchmark нельзя менять без записи причины.
